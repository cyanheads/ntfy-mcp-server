# 2.0.0 — 2026-05-08

Full migration from a hand-rolled MCP server to the [`@cyanheads/mcp-ts-core`](https://www.npmjs.com/package/@cyanheads/mcp-ts-core) framework. Tool surface is being re-ported on this branch — this entry will be expanded as the domain lands.

## Changed

- **Framework** — runs on `@cyanheads/mcp-ts-core` ^0.8.19. Tool, resource, and prompt definitions use the framework builders; handlers throw and the framework catches/classifies/formats.
- **Transports** — stdio plus streamable-HTTP via the framework, configured by `MCP_HTTP_HOST`, `MCP_HTTP_PORT`, `MCP_HTTP_ENDPOINT_PATH`, `MCP_AUTH_MODE`.
- **Config** — env-based, parsed via `parseEnvConfig` in `src/config/server-config.ts`. 1.x env names no longer apply.
- **Logging** — request-scoped `ctx.log` replaces the project's bespoke logger. `MCP_LOG_LEVEL` controls minimum level.
- **Errors** — typed contracts via `ctx.fail(reason, ...)` plus framework error factories (`notFound`, `serviceUnavailable`, …); no project-level `errorHandler`.
- **Engines** — Bun ≥1.3.0, Node ≥24.0.0.

## Removed

- **`smithery.yaml`** — Smithery deploy config dropped during the framework reset.
- **`src/utils/{logger,errorHandler,idGenerator}.ts`** — replaced by framework equivalents on `ctx`.
- **`src/types-global/`** — types collocated with their definitions under the framework.
